# 其他運算服務與選型：Beanstalk、Lightsail、Batch、Outposts

前面認識了 EC2、容器、Fargate、Lambda。這一節補上四個運算服務。

## 你想掌控多少，還是想省事多少

AWS 這一堆運算服務，其實都是同一個問題的不同答案：**「伺服器這些雜事，你想自己管多少？」**

想像你要吃一頓飯：

- 自己買菜、自己煮 → 最自由，想煮什麼都行，但從洗菜到收拾全都你來。這是 **EC2**。
- 肚子餓才叫一份、吃完就走 → 完全不用管廚房、也不用管食材。這是 **Lambda**。

越往「自己來」走，掌控越多、要顧的雜事也越多；越往「別人幫我」走，越省事、但能自己動手的地方也越少。**沒有哪個比較高級，只有哪個適合你現在的情況。** 這一節的四個新服務，都是在這條軸線上、或在旁邊補位的選項。

## 「我只想把應用跑起來」：Beanstalk 與 Lightsail

假設你手上有一個寫好的網站，只想讓它上線，並不想搞懂什麼是負載平衡器、什麼是自動擴展——AWS 有兩個服務就是為這種心情準備的。

**Elastic Beanstalk** 解決的麻煩是：想用 EC2 架一個能正式上線的網頁應用，你得手動把 EC2、負載平衡、自動擴展這些零件一個個拼起來。Beanstalk 讓你**只把程式碼丟上去，底層那些零件它自動幫你佈建好**。之後你還是可以掀開蓋子去調底層（所以掌控權並沒有完全交出去），但預設情況你什麼都不用碰。這種「你顧程式、平台顧基礎設施」的風格，業界叫 PaaS。

**Lightsail** 走得更乾脆。它把一台虛擬伺服器連同**固定、透明的月費打包成方案**，讓你不必面對 AWS 一大堆計價選項。適合簡單網站、小型應用，或第一次碰 AWS、只想用最短路徑把東西跑起來的人。

> 💡 **Beanstalk vs Lightsail 最容易搞混**：兩個都很「省事」，差別在心態——Beanstalk 是「**我有程式碼，你幫我把底層自動佈建起來**」；Lightsail 是「**給我一台簡單、月費透明的小主機就好**」。題目強調「丟程式碼、自動部署」→ Beanstalk；強調「簡單、定價單純、入門」→ Lightsail。

## Batch 與 Outposts

**AWS Batch** 是給「**一次要跑一大批運算工作**」的情況。比如一個晚上要算幾千張影像的渲染、或跑大量科學運算——你不會想一台台手動開機器、排隊、再關掉。Batch 幫你把這些批次工作託管、排程，需要多少算力就給多少，跑完自動收工；底層算力本身跑在 EC2、Fargate 或 EKS 上，Batch 是排程與管理層。

**AWS Outposts** 回答的是**位置**問題，不是省事問題。有些情況——法規要求資料不能離開自家、或對延遲極度敏感——你**必須在自己的機房裡跑**，卻又想沿用 AWS 那一套操作方式與服務。Outposts 就是 AWS 把自家的硬體與服務**實體搬進你的機房**，讓你在地端也能用雲端的方式管理。

## 所有運算選項

```
掌控最多、要管最多 ◄──────────────────────────► 最省事、要管最少
   EC2  →  ECS/EKS on EC2  →  ECS/EKS on Fargate  →  Lambda
```

| 情況 | 選擇 |
|----------|--------|
| 要完全掌控作業系統、長時間執行 | EC2 |
| 已經容器化、自管底層主機 | ECS/EKS（EC2 啟動類型）|
| 已經容器化、不想管底層主機 | ECS/EKS（Fargate）|
| 短小、事件觸發、不想管任何伺服器 | Lambda |
| 要把多個步驟 / 服務串成一個流程 | Step Functions |
| 有程式碼，想讓 AWS 自動處理部署 | Elastic Beanstalk |
| 簡單網站 / 小應用，要最簡單 | Lightsail |
| 一次要跑大量批次運算 | AWS Batch |
| 法規 / 延遲要求，必須在自己機房用 AWS | Outposts |

> 這張表不是拿來背的。先抓住「掌控 vs 省事」這條軸線，前面大半的選項自己就對號入座了；Batch（大量批次）和 Outposts（地端）是軸線外的特例。

## 重點整理

- **Elastic Beanstalk** = 丟程式碼、AWS 自動佈建底層（PaaS）；**Lightsail** = 簡易小主機、定價透明
- Beanstalk（自動佈建底層）容易和 Lightsail（簡單小主機）混淆：用「丟程式碼」vs「一台小主機」來區分
- **AWS Batch** = 大量批次運算
- **Outposts** = 在自己機房裡跑 AWS
