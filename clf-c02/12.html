AI/ML on AWS：用現成的 vs 自己建 ​

AWS 的 AI/ML 服務分成兩大類，：

現成 AI 服務自己建模型的平台
是什麼AWS 預先訓練好的 AI 能力，呼叫 API 就能用讓你從頭建立、訓練、部署自己的模型
需要 ML 知識嗎不用，會呼叫 API 即可需要資料科學 / ML 專業
代表服務Rekognition、Comprehend、Polly…Amazon SageMaker AI
類比即食調理包，拆封即用從食材開始自己下廚，完全客製

「不想碰 ML、直接要結果」→ 用現成 AI 服務

「要用自己的資料訓練專屬模型」→ SageMaker AI。

AWS 現成 AI 服務 ​

這些都是「丟資料進去、拿結果出來」的預訓練服務，記住每個處理哪種資料即可：

服務處理什麼功能重點
Rekognition影像 / 影片物件與場景偵測、人臉分析與比對（可做身分驗證）、影像 OCR、不當內容審核
Comprehend文字（NLP）情感分析、抽取實體 / 關鍵字 / PII 個資
Textract文件 / 表單不只 OCR——還能解析鍵值對與表格結構（發票、申請表自動化）
Transcribe語音 → 文字多語言、講者辨識（diarization）、自動 PII 遮蔽；Transcribe Medical 處理醫療語音
Translate語言神經機器翻譯，支援即時 / 批次；可設自訂術語表
Polly文字 → 語音擬真 / 神經語音；可用 SSML 控語調；適合 IVR、有聲內容
Lex對話打造語音 / 文字聊天機器人（intent / slot；Alexa 同款技術）；常接 Lambda 與 Connect
Personalize推薦即時個人化推薦（Amazon.com 同源技術）
Kendra企業文件搜尋ML 智慧搜尋，用自然語言問內部文件；常當 RAG 資料源
Fraud Detector詐騙偵測託管詐騙 / 假帳號偵測模型

服務名稱幾乎都與其功能有關——Transcribe（謄寫）、Translate（翻譯）、Polly 會說話（像鸚鵡 Polly）、Textract = Text + extract（抽文字）、Comprehend（理解文字）。

生成式 AI on AWS ​

前面的 AI 服務多半是「判斷 / 分類」（這是不是貓？情緒正負？）；生成式 AI 會「創造新內容」——寫文章、生成圖片、寫程式碼。它背後是基礎模型（Foundation Model, FM）：用海量資料訓練出的「博學通才」，可以直接呼叫、也可以再微調。

AWS 的生成式 AI 三層 ​

層級服務做什麼
應用層（現成 AI 助理）Amazon QAWS 的生成式 AI 助理：Q Business（企業知識問答、提升生產力）、Q Developer（寫程式 / 雲端開發助手）
模型層（建生成式 AI 應用）Amazon Bedrock透過 API 使用現成基礎模型（含 Anthropic Claude、Amazon Nova、Meta 等）打造自己的應用——用模型，不訓練模型
基礎建設層（訓練自己的模型）Amazon SageMaker AI要從頭訓練 / 客製自己的模型時用

Amazon Bedrock 詳解 ​

Bedrock 是完全託管、無伺服器的服務，透過單一 API 呼叫多家基礎模型（Anthropic Claude、Amazon Titan / Nova、Meta Llama、Mistral、Cohere 等），不用管任何基礎設施。除了直接呼叫模型，還有四個功能：

功能做什麼
Guardrails（護欄）負責任 AI 安全層——過濾有害內容、PII 個資遮蔽、防提示攻擊（prompt injection）、幻覺偵測
Knowledge Bases（知識庫）託管 RAG：把基礎模型接上你自己的資料（公司文件、S3 等），讓回答有依據
Agents（代理）讓模型自動呼叫 API / 工具，執行多步驟任務
模型客製用自己的資料微調（fine-tuning）現有基礎模型；Model Evaluation 協助比較與選模型

Bedrock = 用現成模型（不建模型），還能「接你的資料（Knowledge Bases）＋加安全護欄（Guardrails）＋讓模型自己辦事（Agents）」。

Amazon Q ​

說明
Q Business企業知識助理，連接公司 40＋ 種資料源（SharePoint、Confluence、S3 等），做問答、摘要、內容生成，提升員工生產力
Q Developer開發 / AWS 助手（整合 IDE、CLI、主控台），程式生成與除錯、AWS 資源問答

Bedrock vs SageMaker ​

Amazon BedrockAmazon SageMaker AI
你做什麼用現成的基礎模型（推論）建立並訓練自己的模型
何時選想快速把生成式 AI 整合進應用、不想自己訓練要用自己的資料訓練專屬模型
類比租一位現成大廚幫你做菜（你只點菜）自己從食材開始下廚

「快速用現成基礎模型 / 生成式 AI」→ Bedrock；「訓練客製模型」→ SageMaker AI。精準的數值預測 / 分類迴歸是傳統 ML 任務，選 SageMaker AI，不是 Bedrock。

重點整理 ​

  • 分類：現成 AI 服務（API 即用、免 ML 知識）vs SageMaker AI（自己建 / 訓練模型）
  • 現成 AI 服務：Rekognition（影像）、Comprehend（文字 NLP）、Textract（文件 / 表單結構，不只 OCR）、Transcribe（語音轉文字）、Polly（文字轉語音）、Translate（翻譯）、Lex（對話機器人）、Personalize（推薦）；Kendra（企業文件智慧搜尋）、Fraud Detector（詐騙偵測）
  • 生成式 AI 三層：Amazon Q（Q Business 企業知識、Q Developer 開發助手）→ Bedrock（用現成基礎模型；加值：Guardrails / Knowledge Bases / Agents）→ SageMaker AI（建 / 訓練自己的模型）
情境需求答案
不想碰 ML、直接要 AI 結果現成 AI 服務（Rekognition / Comprehend / Textract…）
用自己的資料訓練專屬模型SageMaker AI
辨識影像 / 影片裡的物件、人臉Rekognition
分析文字情感 / 抽關鍵字Comprehend
從掃描文件 / 表單抽出文字與表格結構Textract
語音轉文字Transcribe
文字轉語音Polly
打造聊天機器人Lex
個人化推薦Personalize
用自然語言搜尋企業內部文件Kendra
偵測詐騙 / 假帳號Fraud Detector
快速用現成基礎模型建生成式 AI 應用Bedrock
企業員工知識問答 / 提升生產力Amazon Q Business
AI 程式生成 / 開發助手（IDE）Amazon Q Developer
