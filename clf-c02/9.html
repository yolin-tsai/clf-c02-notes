三種儲存類型 ​

AWS 的儲存服務底層有三種方式。

  • 區塊儲存（Block Storage）：資料被切成許多固定大小的區塊，各有自己的位址。改一個大檔案裡的一行字，系統只需要重寫那一行所在的區塊，其他不動。

    這就是區塊儲存「頻繁、小幅修改很有效率」的原因——作業系統和資料庫都跑在區塊儲存上，因為它們無時無刻都在改動小片段。缺點是它像一顆裸硬碟，得先掛載到一台機器、格式化才能用。

  • 物件儲存（Object Storage）：每個檔案是一個完整的物件（檔案本體 + 中繼資料 + 唯一識別碼），放在扁平空間裡。改一行字，得把整個物件重新上傳一次——沒辦法只改一小塊。

    修改這麼「笨重」，為什麼還用它？因為換來兩個巨大好處：幾乎無限的擴展能力，以及透過網址（HTTP）從任何地方存取。適合「寫一次、讀很多次、很少修改」的東西：圖片、影片、備份。

  • 檔案儲存（File Storage）：資料放在階層式資料夾裡，透過網路讓多台機器同時掛載、共享讀寫同一批檔案。就像辦公室的「網路共享磁碟機（NAS）」。

三者對照 ​

區塊（Block）物件（Object）檔案（File）
怎麼存切成固定大小區塊、各有位址整個檔案當一個物件 + 中繼資料階層式資料夾
改一小部分只重寫受影響的區塊（高效）整個物件重新上傳直接改該檔案
怎麼存取掛載到實例、格式化後使用透過 HTTP / API（網址）透過網路掛載，多台同時
AWS 服務EBS、Instance StoreS3EFS、FSx
適合OS、資料庫、頻繁小修改圖片、影片、備份、靜態網站多台共享檔案、內容管理

📌 頻繁小幅修改（資料庫、OS）→ 區塊；寫一次讀很多、海量擴展（媒體、備份）→ 物件；多台機器同時共享同一批檔案 → 檔案。

區塊儲存：Instance Store 與 Amazon EBS ​

EC2 Instance Store（執行個體儲存） ​

實體直接連在 EC2 宿主機上的暫存儲存，速度快，但是是暫時性（Ephemeral）的：

當 EC2 停止或終止，Instance Store 上的資料就永久消失。它綁在那台實體宿主機上，機器一走資料就沒了。只適合放「丟了也沒關係」的資料：快取（Cache）、緩衝（Buffer）、暫存的中間運算結果。

Amazon EBS（Elastic Block Store） ​

網路連接的區塊儲存，像一顆可以掛到 EC2 上的虛擬硬碟。和 Instance Store 最大的不同是它持久（Persistent）：

Instance StoreAmazon EBS
連接方式實體直連宿主機網路連接
持久性暫時（停止 / 終止即消失）持久（資料保留）
可否卸載重掛不行可以
適合快取、緩衝、暫存OS 開機碟、資料庫、要保留的資料

📌 EBS 的限制：EBS 必須和它所掛載的 EC2 在同一個 AZ；一般一顆 EBS 一次服務一台 EC2。

EBS 快照（Snapshots） ​

EBS 的備份機制：某個時間點的狀態備份，自動存放在 AWS 託管的 S3（這是 AWS 後台的儲存空間，不會出現在你自己的 S3 Bucket 清單裡）。

  • 增量備份：第一次完整備份，之後只存「有變動的部分」，省空間又省錢
  • 用途：備份、還原、把磁碟區搬到別的 AZ 或 Region（先建快照，再到目的地從快照建新磁碟區）

†「EBS 綁單一 AZ」的限制靠快照就能突破，它讓你能把資料複製到其他 AZ / Region，是備份與災難復原的基本工具。

物件儲存：Amazon S3 ​

S3（Simple Storage Service）是 AWS 的物件儲存，把檔案（物件）放進儲存貯體（Bucket），透過 HTTP / API 存取。特色是幾乎無限的容量與極高的耐久性（資料被冗餘存放在多個設施）。

適合：網站圖片影片、資料備份與封存、靜態網站託管、巨量資料分析的資料湖。

  • 耐久性（Durability）= 資料不會遺失（存進去之後還在不在）。
  • 可用性（Availability）= 隨時存取得到
  • 耐久性 vs 可用性：S3 Standard 設計為 99.999999999%（11 個 9）耐久性。可用性方面，S3 Standard 是 99.99%。高耐久不代表高可用，兩者要分開看。

S3 儲存類別（Storage Classes）：用存取頻率換成本 ​

儲存類別適合重點
S3 Standard頻繁存取預設，高可用、隨時快速取用
S3 Intelligent-Tiering存取模式不固定 / 不確定AWS 自動在各層間搬移，省心
S3 Standard-IA（不常存取）不常用但偶爾要快速取回儲存較便宜，取回要付費
S3 One Zone-IA不常用、可接受只存單一 AZ更便宜，但只存一個 AZ（較不耐災）
S3 Glacier Instant Retrieval封存、但偶爾要毫秒取回封存價格、即時取回
S3 Glacier Flexible Retrieval封存、可等幾分鐘到數小時更便宜
S3 Glacier Deep Archive長期封存、可等數小時最便宜，適合法規長期保存

📌 頻繁存取 → Standard；不確定多常用、想自動最佳化 → Intelligent-Tiering；長期封存、很少取 → Glacier 系列（取回愈快愈貴）。

S3 Bucket 預設是私有的，且 AWS 有「封鎖公開存取（Block Public Access）」保護。要公開必須刻意關掉這個保護並設權限——很多資料外洩事件就是不小心把 Bucket 設成公開。沒有要公開就維持私有。

S3 生命週期政策（Lifecycle Policy） ​

設定規則，讓物件隨時間自動搬到更便宜的類別：

物件上傳
  → 30 天後自動轉 Standard-IA
  → 90 天後自動轉 Glacier
  → 365 天後自動刪除

很多資料剛上傳時常被存取，過陣子就沒人碰了（月報、Log）。生命週期政策讓系統自動把冷掉的資料移到便宜的層、過期刪除，省下儲存費，不用手動管理。

檔案儲存：Amazon EFS 與 Amazon FSx ​

Amazon EFS（Elastic File System） ​

託管的 NFS 共享檔案系統，給 Linux。最大的價值是多台 EC2 可以跨多個 AZ 同時掛載、共享同一批檔案，容量自動隨資料增減，不用預先配置。

Amazon EBSAmazon EFS
類型區塊檔案（共享）
範圍單一 AZ跨多個 AZ（Region 級）
同時存取一般一台 EC2多台 EC2 同時
容量固定，要手動調整彈性自動伸縮

Amazon FSx ​

當你需要特定的第三方檔案系統時，用 FSx（全託管）：

FSx 型號是什麼適合
FSx for Windows File Server託管的 Windows 共享檔案（SMB 協定、整合 AD）Windows 環境的共享檔案
FSx for Lustre高效能運算（HPC）檔案系統機器學習、大規模運算、需要極高吞吐

EFS 是 Linux / NFS；要 Windows 共享檔案就用 FSx for Windows File Server（EFS 不支援 Windows）。

混合與災難復原儲存 ​

AWS Storage Gateway ​

混合雲儲存：在你的地端機房和 AWS 雲端儲存之間搭一座橋。讓地端應用程式能無縫使用雲端儲存（例如把備份送上雲、把不常用的資料移到雲端），同時在本地保留快取以維持低延遲存取。

AWS Elastic Disaster Recovery（DRS） ​

把你的伺服器持續複製到 AWS，萬一發生災難（機房故障、勒索軟體），能快速在 AWS 上把系統拉起來，把停機時間與資料損失降到最低。

重點整理 ​

  • 三種儲存：區塊（改小片段高效、掛載使用，EBS / Instance Store）、物件（整個物件重傳、網址存取、海量擴展，S3）、檔案（多台共享、資料夾階層，EFS / FSx）
  • Instance Store vs EBS：Instance Store 暫時（停止 / 終止即消失，放快取）；EBS 持久、可卸載重掛、綁單一 AZ
  • EBS 快照：時間點增量備份、存於 AWS 託管的 S3（不在你的 Bucket 清單），可跨 AZ / Region 搬移與復原
  • S3 = 物件儲存、海量；耐久性 99.999999999%（11 個 9）；
  • S3 Bucket 預設私有，公開要刻意設定；耐久性（不遺失）≠ 可用性（存取得到）
  • S3 儲存類別：Standard（頻繁）、Intelligent-Tiering（不確定、自動最佳化）、Standard-IA / One Zone-IA（不常用）、Glacier 三層（封存，取回愈快愈貴）；生命週期政策自動轉類別省錢
  • EBS vs EFS：EBS 單 AZ、單機、區塊；EFS 跨 AZ、多機共享、Linux/NFS、彈性伸縮
  • EFS vs FSx：EFS 是 Linux/NFS；Windows 共享檔案用 FSx for Windows；HPC 用 FSx for Lustre
  • Storage Gateway = 混合雲（地端 ↔ 雲端儲存）
  • Elastic Disaster Recovery = 複製伺服器到 AWS 做災難復原
情境需求答案
資料庫 / OS、頻繁小幅讀寫EBS（區塊、持久）
暫存快取，丟了也沒關係、要最快Instance Store（暫時）
海量圖片影片備份、用網址存取S3（物件）
S3 資料很少存取、長期封存S3 Glacier 系列
不確定多常存取、想自動省錢S3 Intelligent-Tiering
讓 S3 資料隨時間自動轉便宜類別S3 生命週期政策
多台 Linux EC2 跨 AZ 共享檔案EFS
Windows 環境共享檔案FSx for Windows File Server
高效能運算（HPC）檔案系統FSx for Lustre
連接地端與雲端儲存（混合）Storage Gateway
